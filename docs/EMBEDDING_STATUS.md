# Embedding 配置状态

## 当前状态：使用本地 Fallback

由于外部 API 的问题，当前项目使用**本地哈希向量**作为 embedding 方案。

### 配置
```bash
USE_EMBEDDING_API=false
```

### 优缺点

#### ✅ 优点：
- 完全免费，无需 API Key
- 无网络依赖，响应快速
- 开箱即用，无需配置
- 适合开发和测试阶段

#### ❌ 缺点：
- 无法理解语义（只能匹配相同词汇）
- "猫"和"宠物"无法关联
- 记忆检索准确度较低

## 测试的 API 方案

### 1. OpenRouter
- ❌ 需要购买积分（不是免费的）
- 错误：`402 Insufficient credits`

### 2. Hugging Face
- ❌ API 端点已变更，新端点也无法使用
- 错误：`410` → `404 Not Found`

## 建议

### 测试阶段（当前）

**继续使用本地 Fallback**，原因：
1. 可以完成所有功能开发和测试
2. 记忆系统的其他部分（摘要、存储、检索）都能正常工作
3. 只是语义匹配不够精准，但基本功能可用

### 生产阶段（未来）

如果需要更好的语义检索，可以考虑：

1. **购买 OpenRouter 积分**
   - 成本低（约 $0.0001/次）
   - 质量好（OpenAI 模型）
   - 配置简单

2. **自建 Embedding 服务**
   - 使用 sentence-transformers 本地模型
   - 完全免费
   - 需要服务器资源

3. **使用其他免费服务**
   - Cohere（有免费额度）
   - 其他新兴服务

## 实际影响

### 对项目的影响：

#### ✅ 不影响的功能：
- 对话系统 ✅
- 人设系统 ✅
- 会话管理 ✅

- 管理后台 ✅
- 测试评估系统 ✅

#### ⚠️ 受影响的功能：
- 跨会话记忆检索的**精准度**
  - 仍然可以检索，但匹配不够智能
  - 例如："我的猫"和"我的宠物"可能无法关联

### 实际测试结果：

使用本地 fallback 时：
- ✅ 完全相同的词汇：匹配良好
- ⚠️ 同义词/相关词：匹配较差
- ❌ 语义相关但用词不同：基本无法匹配
