# Phase 7 执行计划

最后更新：2026-04-01

## 目标

Phase 7 的目标不是新增功能，而是把当前已经闭环的系统打磨到“稳定、自然、可持续迭代”的状态。

重点包括：

- 对话自然度提升
- 人设一致性提升
- 记忆衔接准确性提升
- 去 AI 味
- 长链路稳定性验证
- 自动化回归固化

---

## 当前前提

当前默认前提如下：

- Phase 1-6 已完成
- 系统代码/API/数据库验收已通过
- 真实浏览器主链路已通过
- 当前允许继续做 Prompt、记忆、展示、小范围逻辑修正
- 当前不进入 Phase 8 新功能开发

---

## 执行原则

### 1. Phase 7 不做大功能扩张

只接受以下类型的改动：

- Prompt 模板调整
- 人设数据调整
- 记忆检索/缓存/展示修正
- 小范围前端交互修正
- 测试工具与评估工具补强

不接受以下类型的改动：

- 新架构迁移
- 大规模 UI 重构
- Phase 8 以后功能提前插入
- 与当前打磨无关的全仓泛化重构

### 2. 每个问题必须先归因再修

每个问题都必须先归到以下四类之一：

- Prompt 问题
- 记忆检索/缓存问题
- 人设数据问题
- 前端展示问题

禁止在没有归因前直接乱改。

### 3. 每轮只做小步收敛

每轮流程固定为：

1. 发现问题
2. 归因
3. 小范围修复
4. 回测相关用例
5. 跑回归
6. 记录结果

---

## 打磨分层

## 第一层：建立基线

### 固定测试人设

至少保留以下测试集合：

- 2 个主测人设
  - 风格差异明显
  - 适合做日常闲聊和情绪场景
- 1 个辅助人设
  - 用于边界和异常场景验证

### 固定测试场景

Phase 7 统一围绕这几类场景进行：

- 日常闲聊
- 情绪安慰
- 记忆衔接
- 人设边界
- 关系推进

### 固定评估维度

人工和自动化统一看以下维度：

- 角色一致性
- 自然度
- 情绪准确性
- 去 AI 味
- 长度适配
- 记忆衔接

---

## 第二层：人工打磨主循环

### 每轮执行内容

每一轮人工测试都要：

- 按固定场景试聊
- 收集明显低分回复
- 记录问题出现的人设、输入、输出、上下文
- 明确预期方向

### 每条问题必须记录

统一记录以下字段：

- 问题编号
- 日期
- 人设
- 分类
- 用户输入
- AI 回复
- 预期回复方向
- 问题归因
- 修复动作
- 回测结果

### 每轮结束的处理方式

- 只修本轮明确复现的问题
- 不顺手扩大范围
- 修完先回测相关 case
- 再跑自动化回归

## 第三层：自动化验证

## 现有自动化工具

当前 Phase 7 默认复用以下工具：

- `npm run test:regression`
- `npm run test:batch`
- `npm run test:prompt-compare`
- `npm run test:auto-eval`
- `npm run test:memory:e2e`
- `npm run test:memory:cache`
- `npm run test:memory:performance`

## 使用规则

### Prompt 或记忆逻辑有变更时

至少跑：

- `npm run typecheck`
- `npm run build`
- `npm test -- --runInBand --no-coverage`
- `npm run test:regression`

### 比较 Prompt 效果时

使用：

- `npm run test:prompt-compare`
- `npm run test:batch`
- `npm run test:auto-eval`

### 记忆链路有改动时

额外跑：

- `npx tsx scripts/test-import-memory-flow.ts`
- `npm run test:memory:e2e`
- `npm run test:memory:cache`

## Phase 7 轮次计划

## 第 1 轮：基础自然度与明显 AI 味清理

目标：

- 先清除最明显的机械感
- 先让对话“像人”

重点检查：

- 回答是否太长
- 是否出现 AI 套话
- 是否像客服/助理
- 是否有明显说教感

优先改动：

- Prompt 模板
- 后处理规则
- 人设 speaking_style / example_dialogues

---

## 第 2 轮：记忆衔接专项

目标：

- 解决“记住了但没用对”
- 解决指代词和跨会话衔接问题

重点检查：

- 导入内容首轮召回
- 跨 session 记忆衔接
- “那个展”“那只猫”“后来呢”这类延续表达
- 同 persona 不同 character 隔离

优先改动：

- retriever
- memory context cache
- 导入后失效刷新
- memory panel 展示逻辑

---

## 第 3 轮：关系感与边界稳定性

目标：

- 让关系推进自然
- 防止过界或突然变味

重点检查：

- 是否忽然变得过度亲密
- 是否忽然变冷、变客服
- 是否在人设边界场景中失真
- 是否会突然暴露系统设定或 AI 身份

优先改动：

- Prompt 关系边界约束
- 人设 default_relationship
- emotional_traits
- quirks

---

## 第 4 轮：长期稳定性测试

目标：

- 验证多轮、多天、多角色下是否稳定

重点检查：

- 长对话后口吻是否漂移
- 长期记忆是否污染
- 管理后台数据是否仍可读
- 指标与真实行为是否一致

优先改动：

- 小范围 bug fix
- 数据清理
- 后台展示降噪

---

## 第 5 轮：回归固化

目标：

- 把已经修好的问题固化成长期测试资产

内容：

- 把典型低分案例加入回归集
- 更新 TEST_CASES
- 更新 baseline
- 固化 prompt compare case

---

## Phase 7 达标标准

满足以下条件后，才视为 Phase 7 可收口：

- 核心回归集稳定通过
- 日常闲聊自然度明显提升
- 不再出现明显 AI 套话
- 导入记忆、跨 session 记忆、character 隔离稳定
- 人设边界场景无明显失真
- 管理后台关键页数据真实且可读
- 真实浏览器长链路测试无新的阻断项

---

## 每轮结束必须产出

每轮打磨结束后必须补一份简短记录，至少包含：

- 本轮测试范围
- 本轮发现问题
- 本轮修复项
- 本轮回归结果
- 是否进入下一轮

建议统一写入：

- `docs/OPTIMIZATION_LOG.md`
