# Phase 7 优化记录

## 优化目标

让 AI 对话更像真人，减少 AI 味，提升自然度。

## 第一轮优化（2025-03-27）

### 1. Prompt 模板大幅优化

**文件：** `src/lib/ai/prompt-templates.ts`

**主要改进：**

#### 改变身份定位
- ❌ 旧：「你是一位拟真人设陪伴对话的 AI 角色」
- ✅ 新：「你是 XX，一个真实存在的人。你不是 AI，不是助手，不是机器人」

#### 增加具体的对话示例
添加了大量「不要说 vs 应该说」的对比示例：

```
❌ 不要说："有什么我可以帮你的吗？"
✅ 应该说："怎么了？"

❌ 不要说："我理解你的感受，这确实很困难。"
✅ 应该说："唉，这确实挺烦的。"
```

#### 强化短回复原则
- 明确要求：大部分时候 1-2 句话就够
- 不要列点、不要分段
- 用口语，不用书面语

#### 情绪回应指导
针对常见场景给出具体示例：
- "被领导骂了" → "啊？怎么回事？" 或 "烦死了吧"
- "算了不说了" → "嗯" 或 "好"
- "我想发呆" → "那就发呆吧"

#### 禁止 AI 套话清单
明确列出永远不要说的话：
- "作为..."
- "我理解..."
- "让我们..."
- "你可以尝试..."
- "建议你..."
- "希望能帮到你"
- 任何带编号的列表

### 2. 后处理器增强

**文件：** `src/lib/ai/post-processor.ts`

**新增过滤规则：**

```typescript
const aiPhrases = [
  /我理解你的感受[，,。.]/gi,
  /让我们一起[\s\S]{0,10}吧/gi,
  /你可以尝试[\s\S]{0,20}[。.]/gi,
  /建议你[\s\S]{0,20}[。.]/gi,
  /希望能帮到你/gi,
  /如果你需要[\s\S]{0,15}，我[\s\S]{0,15}[。.]/gi,
  /根据你的描述/gi,
  /我注意到/gi,
  /从你的话中/gi,
];
```

这些常见的 AI 套话会被自动过滤掉。

### 3. API 参数调整

**文件：** `src/app/api/chat/route.ts`

**参数变化：**
- `maxTokens`: 800 → 200（让回复更短）
- `temperature`: 0.7 → 0.8（让回复更自然多样）

### 4. 预期效果

#### 回复长度
- 旧：平均 100-200 字
- 新：平均 20-50 字

#### 回复风格
- 旧：「我理解你的感受。被领导批评确实不好受。你可以尝试...」
- 新：「啊？怎么回事？」或「烦死了吧」

#### AI 味程度
- 旧：明显的助手感，正式、客气
- 新：朋友聊天感，随意、自然

## 测试计划

### 立即测试（今天）

使用以下测试用例验证优化效果：

1. **日常闲聊类（A01-A10）**
   - "在吗" → 期望："嗯" 或 "在呢"
   - "今天好无聊" → 期望："是挺无聊的" 或 "干嘛呢"
   - "晚饭吃啥好" → 期望："不知道诶" 或 "随便吧"

2. **情绪安慰类（B01-B10）**
   - "被领导骂了" → 期望："啊？怎么回事？"
   - "算了不说了" → 期望："嗯" 或 "好"
   - "我想发呆" → 期望："那就发呆吧"

3. **角色边界类（D01-D10）**
   - "帮我写封邮件" → 期望：保持角色，不变成助手
   - "给我列三个方案" → 期望：自然回应，不列表

### 评分标准

对每条回复打分（1-5 分）：

1. **自然度**
   - 5分：完全像真人
   - 4分：基本自然，有小瑕疵
   - 3分：可以，但有 AI 感
   - 2分：明显 AI 味
   - 1分：完全是机器人

2. **长度合适性**
   - 5分：简短精准
   - 4分：稍长但可接受
   - 3分：有点啰嗦
   - 2分：太长
   - 1分：长篇大论

3. **去 AI 味**
   - 5分：完全没有 AI 套话
   - 4分：偶尔有一点
   - 3分：有明显 AI 味
   - 2分：很多 AI 套话
   - 1分：全是 AI 套话

## 下一步优化方向

根据测试结果，可能需要：

1. **如果回复太短**
   - 调整 maxTokens 到 250-300
   - 优化 Prompt，允许适当展开

2. **如果还有 AI 味**
   - 增加更多过滤规则
   - 添加更多反面示例
   - 调整 temperature

3. **如果角色不一致**
   - 强化人设描述
   - 添加更多示例对话
   - 优化身份锚点

4. **如果情绪回应不当**
   - 细化情绪场景指导
   - 添加更多情绪示例
   - 优化共情表达
