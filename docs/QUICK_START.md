# 🚀 快速开始指南

## 当前配置

✅ **对话模型**：Claude Sonnet 4.5（通过 OpenRouter）
✅ **Embedding**：本地 Fallback（无需 API）
✅ **数据存储**：本地文件（`.data/app-store.json`）

## 启动项目

### 1. 安装依赖（如果还没装）

```bash
cd ai-companion
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

服务器会在 http://localhost:3000 启动

### 3. 登录系统

访问：http://localhost:3000/login

**默认账号：**
- 普通用户：`demo@ai-companion.local` / `demo123456`
- 管理员：`admin@ai-companion.local` / `admin123456`

## 开始 Phase 7：系统打磨

### 准备工作

1. **查看测试用例**
   ```bash
   # 打开测试用例文档
   cat docs/TEST_CASES.md
   ```
   里面有 50 个测试场景

2. **准备测试环境**
   - 创建 2-3 个不同风格的人设
   - 准备记录测试结果

### 测试流程

#### 第一轮：基础功能验证（1-2 天）

**目标：** 确保所有基础功能正常工作

1. 测试 10 个基础对话场景
2. 验证记忆系统是否工作
3. 检查人设一致性
4. 记录所有问题

#### 第二轮：系统性测试（2-3 天）

**目标：** 使用 50 个测试用例全面测试

1. 打开测试面板：http://localhost:3000/admin/testing
2. 逐个测试用例进行测试
3. 对每条回复打分（6 个维度）
4. 记录低分项和问题

**评分标准：**
- 5 分：完美，完全符合预期
- 4 分：很好，有小瑕疵
- 3 分：可以，但有明显问题
- 2 分：较差，需要改进
- 1 分：很差，完全不符合

#### 第三轮：针对性优化（2-3 天）

**根据评分结果优化：**

1. **如果"自然度"低分**
   - 优化 `src/lib/ai/prompt-templates.ts`
   - 加强"防 AI 味"指令
   - 调整 `src/lib/ai/post-processor.ts` 过滤规则

2. **如果"角色一致性"低分**
   - 强化人设的 System Prompt
   - 加入更多示例对话
   - 调整 temperature 参数

3. **如果"记忆准确性"低分**
   - 检查摘要生成质量
   - 调整记忆检索阈值
   - 优化记忆注入方式

4. **如果"情绪回应"低分**
   - 加强情绪感知指令
   - 添加情绪-回应示例
   - 优化情绪检测逻辑

## 常见问题

### Q1: AI 回复太慢？
- 检查网络连接
- OpenRouter 可能有延迟
- 可以调整 `maxTokens` 参数（减少生成长度）

### Q2: AI 忘记之前说的话？
- 检查会话摘要是否生成（需要 4 轮以上对话）
- 访问 `/admin/memories` 查看记忆
- 由于使用本地 embedding，需要用相同词汇

### Q3: AI 回复太像机器人？
- 优化人设的 `speaking_style`
- 加强"防 AI 味"指令
- 添加更多口语化示例

### Q4: 数据存在哪里？
- 本地文件：`.data/app-store.json`
- 包含：用户、会话、消息、记忆、评估数据
- 可以直接编辑（JSON 格式）

### Q5: 如何重置数据？
```bash
# 删除数据文件
rm -rf .data/app-store.json

# 重启服务器，会自动创建新文件
npm run dev
```
